Workflows

Build automation pipelines with directed graphs of steps.

Overview

Workflows are the core building block of Nimbus. A workflow defines a sequence of steps that execute in response to a trigger event. Steps run in order and can branch, wait, or call external services.

Creating a Workflow

const wf = await nimbus.workflows.create({
  name: 'Onboarding',
  trigger: 'user.signup',
  steps: [
    { action: 'send_email', template: 'welcome' },
    { action: 'wait', duration: '1d' },
    { action: 'send_email', template: 'getting-started' },
  ],
});

Listing Workflows

const list = await nimbus.workflows.list();
// Returns an array of workflow objects

Workflow Statuses

StatusDescription
activeRunning and listening for trigger events
pausedTemporarily stopped; no events processed
draftNot yet activated
archivedSoft-deleted; can be restored

Pausing & Resuming

await nimbus.workflows.pause(wf.id);
await nimbus.workflows.resume(wf.id);
Note: Pausing a workflow does not cancel steps already in progress. They will complete before the workflow halts.
