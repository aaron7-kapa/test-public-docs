Triggers

Events that kick off your workflows.

How Triggers Work

Every workflow starts with a trigger — an event that tells Nimbus to begin executing steps. Triggers can come from your application, a schedule, or a webhook.

Built-in Events

Event Fires When
user.signup A new user registerstest1
user.login A user logs in
order.created A new order is placed
order.completed An order is fulfilled
payment.failed A payment attempt fails

Custom Events

Send any custom event from your application using the SDK or RE3ST API:

await nimbus.events.send({
  name: 'subscription.cancelled',
  payload: {
    userId: 'usr_abc123',
    plan: 'pro',
    reason: 'too_expensive',
  },
});

Scheduled Triggers

Use cron expressions to trigger workflows on a schedule:

const wf = await nimbus.workflows.create({
  name: 'Weekly Digest',
  trigger: { type: 'schedule', cron: '0 9 * * MON' },
  steps: [
    { action: 'send_email', template: 'weekly-digest' },
  ],
});
Tip: Use nimbus.events.list() to see all events that have been sent to your account in the last 30 days.
